= Разработка программа хранения нормативной документации =

== Описание работы с документами ==

Библиотека имеет электронные документы. это файлы в формате tif.
Может быть несколько версий данного документа. Файлы хранятся
в каталогах на жестком диске. Ведется бумажный журнал регистрации
документов. Электронные версии документов рассылаются службам. 
Ведется регистрация рассылки документов. Служба идентифицируется 
номером.

== Задача ==

Разработать программу для регистрации и ведения документов.

== Требования ==

1. Журнал регистрации документов (ведения списка документов).
   Он содержит поля:
   1. Номер по порядку (регистрационный номер).
   2. Код документа.
   3. Наименование документа.
   4. Дата введения последней версии. Дата введения в использование?
   5. Кому разослано. Список служб, получивших последнюю? копию документа.
      Может это лишнее в журнале?
2. Поиск документов по наименованию, по буквенно-цифровому коду,
   по дате введения.
4. Печать списка документов с выбором полей для печати.
5. Хранение всех версий документа.
   Версия документа определяется датой введения в использование и
   файлом документа.


== Анализ ==

Сущности

1. Документ.
2. Файл документа.
3. Журнал.
4. Версия документа.
5. Служба.

Можно ввести понятие регистрационной карточки документа. Она содержит данные
регистрации документа и ведения его жизненного цикла. Журнал будет содержать
карточки документов.

1. Карточка документа. 
   Атрибуты:
   1. Регистрационный номер.
   2. Код документа.
   3. Наименование документа.
   4. Версии документов.
2. Версия документа.
   Атрибуты:
   1. Дата введения.
   2. Список получивших.
   2. Файл.
3. Файл.
   Атрибуты:
   1. MIME тип.
   2. содержимое.
4. Журнал.
   Атрибуты:
   1. Список карточек документов.
5. Служба.
   Атрибуты:
   1. Наименование.
   2. Идентификационный номер.
